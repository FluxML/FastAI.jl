Finetuning a pretrained model

To access pretrained computer vision models, you’ll have to install a work-in-progress branch of Metalhead.jl as detailed on the setup page.

using FastAI, CairoMakie, Metalhead
import CairoMakie

Let’s load the image classification dataset ImageNette. You’re free to replace this by any of the other classification datasets in FastAI.DATASETS.

DATASETNAME = "imagenette2-160";

Next we load the dataset and create a learning method, same as on the quickstart page.

datas = Datasets.loadfolderdata(
    datasetpath("imagenette2-160"),
    filterfn=isimagefile,
    loadfn=(loadfile, parentname),
    splitfn=Datasets.grandparentname)

traindata, validdata = datas["train"], datas["val"]
classes = unique(eachobs(traindata[2]))

method = BlockMethod(
    (FastAI.Image{2}(), FastAI.Label(classes)),
    (
        ProjectiveTransforms((128, 128)),
        ImagePreprocessing(),
        OneHot()
    )
)
BlockMethod(FastAI.Image{2} -> FastAI.Label{String})

Let’s check that the data looks alright:

xs, ys = makebatch(method, traindata, 1:9)
plotbatch(method, xs, ys)

Now we load a pretrained model. Through indexing, we cut off the last 3 layers since we want to put a new classification head on the convolutional feature extractor, the so-called backbone.

backbone = Metalhead.ResNet50(pretrain=true).layers[1:end-3];

We pass it to methodlearner which will call methodmodel to stack a classification head on top of the backbone:

learner = methodlearner(
    method,
    traindata,
    backbone,
    ToGPU(), Metrics(accuracy);
    validdata=validdata)
Learner()

The fine-tuning itself is done with finetune!. It follows the same protocol as the fastai implementation.

In short, it freezes the backbone portion of the model and trains for one epoch so that the newly initialized classification head can train for a bit. Afterwards backbone and head are trained for a number of epochs, with the backbone having a lower learning rate.

finetune!(learner, 4)
